# Ruchi Realty — Website

A small **2-page React site**. There is **no build step**: React, ReactDOM and Babel
are loaded from a CDN, and the `.jsx` files are compiled **in the browser** at page load
(`<script type="text/babel">`). To view it, just open an HTML file in a browser
(ideally through a tiny local server so the `.jsx` files load — see below).

## Pages

| File | Page |
|------|------|
| `Homepage.html` | Home |
| `Projects.html` | All Projects (filterable catalogue) |
| `Admin.html` | Dummy admin panel preview (localStorage-backed) |

Both pages pull the same source from `js/` and `css/`.

## Folder layout

```
Homepage.html          ← page 1
Projects.html          ← page 2
js/                    ← React source (this is where you work)
  data.jsx               project list + site content (edit copy/projects here)
  components.jsx         shared bits (RImg, Reveal, Icon, …)
  sections1.jsx          Nav + header mega-menu, Hero, Intro, Projects strip
  sections2/3/4.jsx      the rest of the homepage sections + Footer
  projects-page.jsx      the All Projects page (filters)
  app.jsx                homepage entry (mounts React)
  tweaks-panel.jsx       dev-only design tweak panel
css/
  tokens.css             colours, fonts, spacing variables
  site.css               all component styles
assets/  uploads/  fonts/   images & fonts used by the site
build/                 ← generated artifacts (not edited by hand)
  homepage-offline.html      single self-contained file (everything inlined) for sharing offline
  homepage-standalone-src.html   the template the offline file is built from
archive/               ← tooling leftovers, referenced by nothing
  _ds/                   exported design-system reference
  screenshots/           review screenshots
  .thumbnail             preview thumbnail
```

## Viewing locally

Because pages load `.jsx` over `fetch`, opening via `file://` can be blocked by the
browser. Run any static server from this folder, e.g.:

```
python3 -m http.server 8000
```

then visit `http://localhost:8000/Homepage.html`.

Admin preview:

- URL: `http://localhost:8000/Admin.html`
- Dummy login: `admin@ruchirealty.local` / `admin123`
- Data is stored in browser localStorage for review only.
- Image upload controls store local preview images in browser storage until real upload credentials are connected.
- Final Supabase SQL is in `backend/sql/00_complete_setup.sql`.

## Editing

- **Project cards / copy:** `js/data.jsx`
- **Header / nav / hero / projects strip:** `js/sections1.jsx`
- **Other homepage sections / footer:** `js/sections2.jsx`, `sections3.jsx`, `sections4.jsx`
- **All Projects page & filters:** `js/projects-page.jsx`
- **Dummy backend/admin bridge:** `js/backend-client.jsx`, `js/admin-panel.jsx`, `Admin.html`
- **Colours / type / spacing:** `css/tokens.css` · **component styles:** `css/site.css`

## Notes on `build/`

- `homepage-offline.html` is a **frozen snapshot** generated by a bundler; it does not
  pick up later edits to `js/`/`css/`. Regenerate it from `homepage-standalone-src.html`
  when you need a fresh single-file copy.
- Nothing in `build/` or `archive/` is needed to run the live site — only the root pages,
  `js/`, `css/`, `assets/`, `uploads/` and `fonts/` are.
